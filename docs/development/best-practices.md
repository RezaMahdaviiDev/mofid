# نکات مهم در پرامت وایب کدیبنگ

## نکات اصلی از درخواست شما:

### 1. ساختار پروژه از کوچک به بزرگ
- همیشه از اجزای کوچک شروع کنید
- ابتدا هسته اصلی را بسازید
- سپس به تدریج قابلیت‌ها را اضافه کنید
- از ساختار ماژولار استفاده کنید

### 2. کار تدریجی و قدم به قدم
- هرگز همزمان روی چند پروژه بزرگ کار نکنید
- هر مرحله را کامل کنید و تست کنید
- بعد از تأیید هر مرحله، به مرحله بعد بروید
- از پیچیدگی‌های غیرضروری پرهیز کنید

### 3. مدیریت فایل‌های تست
- فایل‌های تست موقت ایجاد کنید
- بعد از اتمام تست، فایل‌های تست را پاک کنید
- از فایل‌های تست برای اعتبارسنجی استفاده کنید
- محیط کد اصلی را تمیز نگه دارید

## نکات اضافی از من:

### 4. مستندسازی مناسب
- کد خود را به خوبی مستند کنید
- از کامنت‌های معنادار استفاده کنید
- ساختار پروژه را در فایل README توضیح دهید
- تغییرات مهم را ثبت کنید

### 5. مدیریت وابستگی‌ها
- وابستگی‌های پروژه را مدیریت کنید
- از نسخه‌بندی مناسب استفاده کنید
- وابستگی‌های غیرضروری را حذف کنید
- محیط توسعه را ایزوله نگه دارید

### 6. تست و اعتبارسنجی
- تست واحد برای هر ماژول بنویسید
- تست یکپارچگی انجام دهید
- تست عملکردی داشته باشید
- از ابزارهای تست خودکار استفاده کنید

### 7. مدیریت خطاها
- خطاها را به درستی مدیریت کنید
- پیام‌های خطای معنادار ارائه دهید
- لاگ‌گیری مناسب داشته باشید
- از crash کردن برنامه جلوگیری کنید

### 8. امنیت
- ورودی‌های کاربر را اعتبارسنجی کنید
- از تزریق کد جلوگیری کنید
- احراز هویت مناسب پیاده‌سازی کنید
- داده‌های حساس را محافظت کنید

### 9. بهینه‌سازی
- کد را بهینه بنویسید
- از منابع سیستم به درستی استفاده کنید
- کش‌گذاری مناسب داشته باشید
- عملکرد را مانیتور کنید

### 10. نگهداری و توسعه
- کد را قابل نگهداری بنویسید
- از الگوهای طراحی مناسب استفاده کنید
- refactoring منظم انجام دهید
- استانداردهای کدنویسی را رعایت کنید

## نمونه ساختار پروژه:

```
پروژه-اصلی/
├── src/
│   ├── core/           # هسته اصلی
│   ├── modules/        # ماژول‌ها
│   ├── utils/          # ابزارهای کمکی
│   └── config/         # تنظیمات
├── tests/              # تست‌ها
├── docs/               # مستندات
└── README.md           # راهنمای پروژه
```

## فرآیند توسعه:

1. **تحلیل نیازها** - نیازمندی‌ها را مشخص کنید
2. **طراحی معماری** - ساختار کلی را طراحی کنید
3. **پیاده‌سازی هسته** - بخش اصلی را بسازید
4. **تست و اعتبارسنجی** - عملکرد را بررسی کنید
5. **اضافه کردن قابلیت‌ها** - ویژگی‌های جدید اضافه کنید
6. **بهینه‌سازی** - عملکرد را بهبود دهید
7. **مستندسازی** - کد و فرآیند را مستند کنید

این رویکرد تضمین می‌کند که پروژه شما به صورت کنترل‌شده و با کیفیت بالا توسعه یابد.
